# ByteWhere Content

This repository contains the public writing, ideas, and experiments coming out of **ByteWhere Labs**.

These posts are not marketing.
They are build logs, design thinking, mistakes, and direction from someone actively building systems.

If you’re looking for polished hype, this isn’t that.
If you’re interested in how things are actually made — this is.

---

## What This Repo Is

* Blog posts written for builders
* Architecture thinking in public
* Experiments around AI, systems, kernels, automation, and tooling
* Notes from projects like:

  * ForgeKernel
  * ByteForce
  * Pyact
  * AI-native infrastructure

Everything here is written to be useful, honest, and practical.

---

## What This Repo Is Not

* Not a product repo
* Not a tutorial series
* Not marketing copy
* Not documentation for a finished system

This is thinking while building.

---

## Structure

```
posts/        # All published MDX posts
topics.md     # Backlog of future ideas
assets/       # Images or diagrams used in posts
```

Posts are written in MDX so they can be used by modern static site generators.

---

## Philosophy

* Build first, explain second
* No fake certainty
* No hype without proof
* Mistakes are part of the log
* Direction matters more than perfection

If something changes, the writing will reflect that.

## How Content Is Made

Posts are produced through a private content engine that:

* Drafts from structured prompts
* Enforces a builder-first style
* Supports review before publish

Only the **content** is public.
